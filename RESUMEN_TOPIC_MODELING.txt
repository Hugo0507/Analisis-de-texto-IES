================================================================================
                        ALGORITMOS IMPLEMENTADOS
================================================================================

Se implementaron 3 algoritmos complementarios de Topic Modeling:

┌────────────────────────────────────────────────────────────────────────────┐
│ 1. LDA (Latent Dirichlet Allocation)                                      │
│    ✓ Modelo probabilístico generativo                                     │
│    ✓ Interpreta documentos como mezcla de temas                           │
│    ✓ Cada tema como distribución de palabras                              │
│    ✓ Métrica: Perplejidad, Log-likelihood                                 │
│    ✓ Vectorización: Bag of Words (CountVectorizer)                        │
└────────────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────────────┐
│ 2. NMF (Non-negative Matrix Factorization)                                │
│    ✓ Modelo no probabilístico (algebraico)                                │
│    ✓ Factorización de matrices no negativas                               │
│    ✓ Temas más interpretables (sin valores negativos)                     │
│    ✓ Métrica: Reconstruction Error                                        │
│    ✓ Vectorización: TF-IDF                                                │
└────────────────────────────────────────────────────────────────────────────┘

┌────────────────────────────────────────────────────────────────────────────┐
│ 3. LSA (Latent Semantic Analysis)                                         │
│    ✓ Modelo no probabilístico (SVD)                                       │
│    ✓ Descomposición en valores singulares truncada                        │
│    ✓ Captura relaciones semánticas latentes                               │
│    ✓ Métrica: Explained Variance                                          │
│    ✓ Vectorización: TF-IDF                                                │
└────────────────────────────────────────────────────────────────────────────┘

================================================================================
                        FUNCIONALIDADES DEL MÓDULO
================================================================================

CLASE: TopicModelingAnalyzer
-----------------------------

1. prepare_documents(texts_dict)
   → Prepara documentos para modelado

2. fit_lda(documents, n_topics=10, ...)
   → Entrena modelo LDA
   → Retorna: temas, distribuciones, perplejidad, log-likelihood

3. fit_nmf(documents, n_topics=10, ...)
   → Entrena modelo NMF
   → Retorna: temas, distribuciones, reconstruction_error

4. fit_lsa(documents, n_topics=10, ...)
   → Entrena modelo LSA
   → Retorna: temas, distribuciones, explained_variance

5. _extract_top_words(topic_word_dist, feature_names, n_words=10)
   → Extrae top palabras de cada tema

6. get_document_topics(doc_topic_dist, doc_names, threshold=0.1)
   → Obtiene temas dominantes por documento

7. compare_models(lda_results, nmf_results, lsa_results)
   → Compara resultados de los 3 modelos
   → Calcula overlap de vocabulario (Jaccard similarity)

8. analyze_topic_evolution(doc_topics, doc_names)
   → Analiza evolución de temas en el corpus

VALIDACIÓN:
  • Se genera hash MD5 de configuración
  • Incluye: n_topics, max_features, min_df, max_df, num_docs
  • Cualquier cambio invalida caché

                        FORMATO DE CSV EXPORTADOS
================================================================================

1. MATRICES DOCUMENTO-TEMA (lda_matrix.csv, nmf_matrix.csv, lsa_matrix.csv)

   Formato:
   ```
               Tema 1  Tema 2  Tema 3  ...
   documento1   0.25    0.15    0.05   ...
   ```

   - Cada fila: 1 documento

2. TOP PALABRAS POR TEMA (*_topic_words.csv)

   Formato:
   ```
   Tema,Palabra,Peso
   Tema 1,digital,0.234
   Tema 1,transformation,0.189
   Tema 2,student,0.198
   ```

   Uso:
   - Interpretar qué representa cada tema
   - Visualizar nube de palabras por tema
   - Análisis semántico

EJEMPLO 1: Clustering de Documentos
------------------------------------
```python
import pandas as pd
from sklearn.cluster import KMeans

# Cargar matriz LDA
lda_matrix = pd.read_csv('lda_matrix.csv', index_col=0)

# Clustering
kmeans = KMeans(n_clusters=5)
clusters = kmeans.fit_predict(lda_matrix)

print(f"Cluster assignments: {clusters}")
```

EJEMPLO 2: Similitud entre Documentos
--------------------------------------
```python
from sklearn.metrics.pairwise import cosine_similarity

# Similitud
similarity = cosine_similarity(lda_matrix)
similarity_df = pd.DataFrame(similarity,
                              index=lda_matrix.index,
                              columns=lda_matrix.index)

# Documentos más similares
doc = 'documento1.txt'
similar = similarity_df[doc].sort_values(ascending=False)[1:6]
print(similar)
```

================================================================================
                          MÉTRICAS DE CALIDAD
================================================================================

LDA:
  • Perplejidad:
    - Qué tan bien el modelo predice datos no vistos
    - MENOR = MEJOR
    - Típico: 100-1000 (depende del corpus)

  • Log-likelihood:
    - Ajuste del modelo a los datos
    - MAYOR (menos negativo) = MEJOR

NMF:
  • Reconstruction Error:
    - Error al aproximar la matriz original
    - MENOR = MEJOR
    - Típico: 10-100

LSA:
  • Explained Variance:
    - Porcentaje de información capturada
    - MAYOR = MEJOR
    - Típico: 30%-70%

OVERLAP DE TEMAS:
  • Jaccard Similarity entre pares de modelos
  • 0.0 = Vocabularios completamente diferentes
  • 1.0 = Vocabularios idénticos
  • Típico: 0.2-0.5
